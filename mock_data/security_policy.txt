NovaMed Systems — Information Security Policy
Version 3.2 | Last Updated: January 2025

1. HIPAA Compliance & Certifications

NovaMed Systems maintains full compliance with the Health Insurance Portability and Accountability Act (HIPAA) and the HITECH Act. The platform undergoes annual HIPAA audits conducted by a certified third-party auditor. NovaMed holds SOC 2 Type II certification, renewed annually, covering security, availability, and confidentiality trust service criteria. NovaMed is also ISO 27001 certified as of March 2024.

2. Data Encryption

All patient data is encrypted at rest using AES-256 encryption. Data in transit is secured using TLS 1.3. Encryption keys are managed via AWS Key Management Service (KMS) with automatic rotation every 90 days. Database-level encryption is enabled for all customer schemas.

3. Access Controls

NovaMed implements Role-Based Access Control (RBAC) across all platform modules. User roles include: System Administrator, Clinical Staff, Billing Staff, Auditor, and Read-Only. All role assignments are reviewed quarterly by the customer's designated security officer. Principle of least privilege is enforced at the API and database levels. Access logs are retained for 7 years.

4. Authentication

The platform supports Single Sign-On (SSO) via SAML 2.0 and OAuth 2.0, compatible with major identity providers including Okta, Azure AD, and Google Workspace. Multi-Factor Authentication (MFA) is mandatory for all administrative accounts and optional (but strongly recommended) for clinical staff accounts. Passwordless authentication via hardware security keys (FIDO2) is available for enterprise customers. Session tokens expire after 8 hours of inactivity.

5. Incident Response

NovaMed maintains a documented Incident Response Plan (IRP) reviewed annually. In the event of a suspected data breach, the security team is notified within 1 hour of detection. Affected customers are notified within 48 hours of confirmed breach. Regulatory bodies (HHS) are notified within 60 days as required by the HIPAA Breach Notification Rule. All security incidents are logged and reviewed in post-incident reports shared with affected customers.

6. Employee Data Access

NovaMed employees access customer data only on a need-to-know basis. All access is logged to an immutable audit trail retained for 7 years. Background checks are conducted on all employees handling Protected Health Information (PHI). Access by NovaMed support staff requires customer approval via a formal access request workflow. Privileged access management (PAM) tools enforce time-limited, session-recorded access for infrastructure engineers.

7. Third-Party Data Sharing

NovaMed does not sell or share patient data with third parties for marketing or analytics purposes. Data is shared with third parties only under Business Associate Agreements (BAAs) when required for service delivery (e.g., cloud providers, email delivery). A current list of sub-processors is available upon request and maintained at our trust portal at trust.novamed.io.

8. Security Testing

Penetration testing is conducted bi-annually by a certified external security firm (most recently CrowdStrike Red Team, Q4 2024). Automated vulnerability scanning runs continuously on all production systems using AWS Inspector and Qualys. Static application security testing (SAST) and dynamic application security testing (DAST) are integrated into the CI/CD pipeline. Security findings are tracked and remediated according to severity SLAs: Critical within 24 hours, High within 7 days.

9. Data Portability

Upon contract termination, customers may request a full export of their data in HL7 FHIR R4 format. Data export requests are fulfilled within 30 business days. Data is retained for 90 days post-termination and then securely deleted. A Certificate of Destruction is provided upon request. No patient data is retained beyond 90 days post-termination without explicit written consent.
